# 微信云开发部署与体验版手册

本文既记录当前 `0.2.0` 的部署结果，也提供不包含环境 ID、账号和密钥的重复部署清单。

## 1. 当前部署快照

截至 2026-07-17：

- `0.2.0` 已上传并配置为团队体验版，入口为 `pages/launch/launch`。
- 7 个 Node.js 普通云函数已经部署并在线。
- 4 个数据库集合、2 条种子配置和 1 个组合索引已经创建。
- 四个集合均设置为客户端不可读写，只允许管理端和云函数操作。
- 云函数调用规则为已认证用户可调用：`{ "*": { "invoke": "auth != null" } }`。
- 云存储使用免费环境提供的“仅创建者可读写”预设权限。
- 当前环境只用于联调和体验版，未提交正式审核、未正式发布。

仓库不记录实际环境 ID、云文件地址、OpenID、账号、密钥、开发者工具端口或控制台截图。

## 2. 环境预检

1. 使用具备项目开发权限的微信账号打开微信开发者工具。
2. 确认工具读取仓库中的 `project.config.json`，不要在文档或业务代码中重复写 AppID。
3. 如需命令行部署，在开发者工具“设置 → 安全设置”中临时开启服务端口；端口属于本机配置，不进入仓库。
4. 打开云开发控制台，确认目标环境关联当前项目，并记录套餐到期时间到团队的私有运维记录。
5. 在开发者工具中把 `cloudfunctions/` 的当前环境指向目标环境；小程序端继续使用动态默认环境。

## 3. 数据库配置

创建以下集合：

- `users`
- `detection_records`
- `app_config`
- `model_configs`

配置步骤：

1. 四个集合全部选择“所有用户不可读写”。
2. 向 `app_config` 导入 `cloudbase/seeds/app-config.json`。
3. 向 `model_configs` 导入 `cloudbase/seeds/model-config.json`。
4. 为 `detection_records` 创建非唯一组合索引：`_openid` 升序、`createdAt` 降序。
5. 检查 `app_config/global` 保持 `mode: "demo"`。
6. 检查 `model_configs/future-model-v1` 保持禁用，不得用于真实分析。

## 4. 存储与云函数权限

### 免费体验环境

- 数据库：四个集合使用“所有用户不可读写”。
- 云存储：选择“仅创建者可读写”。免费环境若提示自定义存储规则需要升级，不要为了体验版切换套餐。
- 云函数：应用 `cloudbase/rules/functions-authenticated-only.json` 中的已认证用户调用规则。

数据库和存储规则只是第一道边界。所有业务云函数仍必须使用微信上下文中的 OpenID 校验身份、查询条件、资源所有权和文件路径。

### 正式发布前

升级到适合生产的付费套餐后，重新评估并应用显式资源规则。`cloudbase/rules/storage-owner-only.json` 是云存储自定义规则模板，不能视为当前免费环境已经应用的配置。任何规则变更都要在两个测试账号下重新验证隔离。

## 5. 云函数部署

依次部署：

1. `login`
2. `analyzeDemo`
3. `createDetection`
4. `listDetections`
5. `getDetection`
6. `deleteDetection`
7. `getAppConfig`

每个函数使用 Node.js 18.15，并选择“上传并部署：云端安装依赖”。不要提交或上传本地 `node_modules`。

部署后先验证启动登录，再走通：

```text
选择真实图片
  → 云端演示分析
  → 上传并保存记录
  → 历史分页
  → 私有图片详情
  → 删除图片与记录
```

还需使用第二个测试账号确认无法获取或删除第一个账号的记录和图片。

## 6. 上传体验版

1. 执行语法、JSON、自动化测试和敏感信息检查。
2. 在微信开发者工具中普通编译，确认控制台无项目代码错误。
3. 上传版本并填写准确的功能备注。
4. 在小程序管理后台设为体验版，入口使用 `pages/launch/launch`。
5. 通过成员管理添加体验成员；体验版不等于正式审核或发布。
6. 在至少两种 Android 和一种 iOS 设备检查图片选择、保存、历史、删除和安全区。

当前已上传版本为 `0.2.0`，备注为“接入微信云开发身份、云存储、云端历史与演示分析”。

## 7. 发布前检查

- [ ] 升级生产套餐并确认到期时间、配额和预算。
- [ ] 拆分开发和生产环境，环境标识只保存在团队私有配置中。
- [ ] 复核数据库、存储、云函数权限及双账号隔离。
- [ ] 配置云函数错误率、调用量、数据库、存储和预算告警。
- [ ] 制定备份、恢复、回滚和数据修复流程。
- [ ] 确认图片与记录保存期限、隐私说明、免责声明和删除机制。
- [ ] 完成多机型、弱网、失败重试和回归测试。
- [ ] 只有真实算法经过科研验收后，才能移除演示标识并提交相应版本审核。

## 8. 免费环境边界

免费环境仅用于当前联调和体验版，不作为长期生产环境。不要启用云托管、预置并发或其他非当前原型所需的付费能力。正式发布前应升级套餐并复查当时的官方计费和有效期规则。
