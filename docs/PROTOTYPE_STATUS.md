# GlowLetter 原型演示版阶段总结与后续计划

> 版本：`0.2.0`
>
> 阶段状态：微信小程序体验版已部署成功，尚未提交正式审核
>
> 更新日期：2026-07-17

## 1. 阶段结论

GlowLetter 已经从前端原型发展为可供团队体验的微信小程序：用户可以完成启动、填写样品、选择图片、观看分析、保存结果、查看历史与详情、删除记录的完整流程。小程序已经接入微信云开发，部署了访客身份、演示分析、云数据库、私有图片存储和按用户隔离的历史记录。

当前交付是“云端原型演示版”，不是“真实检测产品”。服务端返回固定、可复现的演示结果，所有结果始终包含 `isDemo: true`。目前不能据此判断真实萘浓度、污染等级或处理方案。

## 2. 已完成交付

### 小程序端

- 完成 10 个产品视图和自定义导航、安全区、转场与状态适配。
- 建立身份、配置、记录、存储服务层和图片、分析适配器。
- 云端初始化失败时提供重试；只有用户主动选择才进入离线演示。
- 分析动画至少展示约 1.6 秒，并等待云函数成功后进入结果页。
- 真实图片在保存记录时上传，内置演示图不重复占用云存储。
- 云端历史支持分页、下拉刷新、详情、删除和私有图片临时地址。
- 本地缓存只保存草稿、页面间结果和显式离线记录，不混入云端历史。

### 云端

- 部署 7 个普通云函数：`login`、`analyzeDemo`、`createDetection`、`listDetections`、`getDetection`、`deleteDetection`、`getAppConfig`。
- 创建 `users`、`detection_records`、`app_config`、`model_configs` 四个集合。
- 初始化公开演示配置和禁用的未来模型占位配置。
- 创建 `_openid` 与 `createdAt` 组合索引。
- 数据库仅允许管理端和云函数操作；云函数只允许认证用户调用。
- 云存储在免费环境中使用“仅创建者可读写”预设权限。
- 云函数使用服务端 OpenID 做查询与删除所有权校验，不信任前端身份或结果字段。

### 部署与工程质量

- `0.2.0` 已上传并配置为团队体验版，启动入口为 `pages/launch/launch`。
- 小程序包经过资源优化，上传大小约 1.1 MB。
- 已建立 9 项自动化测试，覆盖演示模式、结果防篡改、文件路径、分页、删除异常、内置图免上传、孤儿图片清理和离线历史。
- 已建立 JavaScript 语法、JSON、差异格式和敏感信息检查流程。
- 仓库未保存云环境 ID、密钥、OpenID、开发工具端口、用户路径或账号配置。

## 3. 当前可以演示什么

1. 微信访客身份初始化和重复启动的幂等行为。
2. 使用内置图或本地图片完成云端演示分析。
3. 上传真实图片并保存当前用户的演示记录。
4. 分页查看云端历史，按需查看私有图片。
5. 删除记录及其关联图片。
6. 云端不可用时由用户主动进入独立的离线演示模式。

## 4. 当前不能承诺什么

- 不能输出真实萘浓度或科学有效的污染等级。
- 没有实现孔板定位、图像质量判断、背景校正或荧光特征计算。
- 没有接入真实标准曲线、换算公式、阈值、复测规则和处理建议。
- 没有管理员后台、批量数据管理、报告导出和审计能力。
- 免费环境不适合作为长期生产环境，体验版也不等于正式发布。
- 两账号隔离和多机型真机矩阵仍需团队完成最终人工验收。

## 5. 实现真实功能需要的输入

### 实验与科学输入

| 需要 | 最低交付内容 | 用途 |
| --- | --- | --- |
| 原始图像数据集 | 原图、拍摄条件、样品编号、真实标签和异常样本 | 开发与独立验证图像分析 |
| 孔板定义 | 96 孔坐标、行列方向、分组、空白/标准/对照孔 | 定位孔位和建立校正基准 |
| 标准曲线 | 曲线形式、参数、单位、有效范围、校准批次和版本 | 将荧光特征换算为浓度 |
| 分级规则 | 各等级阈值、临界值、超范围与复测规则 | 生成可解释结果状态 |
| 验收指标 | 重复性、允许误差、对照要求、人工复核和签字人 | 判断真实模型是否可以发布 |
| 结果文案 | 经实验和合规确认的建议、免责声明 | 防止误导用户 |

### 硬件与采集输入

- 黑箱、摄像头和光源型号及固定方式。
- 曝光、焦距、距离、方向、环境光和拍摄时序规范。
- 图片从硬件到小程序的传输方式、格式、尺寸和失败处理。
- 可用于判断曝光、清晰度、孔板完整性和方向的质量标准。

### 产品与合规输入

- 目标用户、使用场景、正式产品名称和小程序类目。
- 原始图片与检测记录的保存期限、删除政策和用户告知。
- 真实结论的责任边界、免责声明和异常结果处理流程。
- 是否需要管理员、报告导出、分享、审计和团队协作功能。

### 工程与运维输入

- 正式发布时间、生产负责人、云资源预算和告警接收人。
- 开发、测试、生产环境的分离和访问成员名单。
- 真实分析接口的输入、输出、错误码、超时、幂等和版本策略。
- 两个测试微信账号，以及至少两种 Android 和一种 iOS 验收设备。

## 6. 推荐后续里程碑

### M1：规格冻结

收齐实验数据、硬件规范、孔位配置、公式、阈值和验收指标，形成带版本号的数据字典与接口协议。此阶段不改为真实模式。

### M2：离线算法验证

在不影响小程序体验版的独立分析代码中完成图像质量检查、孔板定位、特征提取和浓度换算。使用训练/调试集与独立验证集，输出指标和失败样本报告。

### M3：云端真实分析联调

把通过验证的分析实现封装为新的云端能力，通过配置开关与 `analyzeDemo` 隔离。记录模型版本、输入摘要和失败原因，保留可回退的演示路径。

### M4：科研与产品验收

由实验负责人确认曲线、误差、阈值和文案；完成双账号隔离、多设备、弱网、异常、重复提交、删除和回归测试。只有验收签字后才允许真实结果进入用户界面。

### M5：生产发布

升级付费套餐，拆分生产环境，配置最小权限、备份、监控、配额和预算告警。完成隐私说明、数据留存、审核材料和回滚预案后，提交微信正式审核。

## 7. 发布门槛

在以下条件全部完成前，项目应继续保持体验版和 `demo` 标识：

- 科学公式、阈值、有效范围和验收指标有明确版本与负责人。
- 真实数据独立验证通过，并有可复现的结果记录。
- 双账号私有数据隔离和多机型真机测试通过。
- 生产环境、付费套餐、监控、备份、告警和回滚准备完成。
- 隐私、留存、删除、免责声明和小程序审核材料确认完成。
