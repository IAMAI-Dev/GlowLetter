# 开发计划

> 最后更新：2026-07-17
>
> 当前里程碑：`0.2.0` 原型演示版已部署为体验版；下一里程碑是真实检测能力定义与验证。

## Phase 0：项目与原型整理（已完成）

- [x] 建立原生微信小程序入口、全局样式与本地演示数据层。
- [x] 将 OpenDesign 资料整理到被 Git 忽略的 `opendesign/`。
- [x] 建立项目开发规范和 Conventional Commits 流程。

## Phase 1：界面与本地演示（已完成）

- [x] 转换启动、首页、样品信息、图片选择、分析、结果、历史、详情、项目说明和“我的”10 个产品视图。
- [x] 实现自定义顶部导航与悬浮胶囊底部导航，不使用原生 `tabBar`。
- [x] 将首页、历史、“我的”合并为常驻主页面壳层并保留滚动位置。
- [x] 增加统一页面转场、加载、空数据、失败和重试状态。
- [x] 实现草稿、内置演示图、分析动画、结果保存和离线历史流程。

## Phase 2：微信云开发原型（已完成）

- [x] 使用现有单个免费环境完成联调，环境标识不写入仓库。
- [x] 建立 `services/` 与 `adapters/` 边界，页面不直接访问数据库。
- [x] 部署 `login`、`analyzeDemo`、`createDetection`、`listDetections`、`getDetection`、`deleteDetection`、`getAppConfig` 7 个云函数。
- [x] 创建 `users`、`detection_records`、`app_config`、`model_configs` 四个集合及种子数据。
- [x] 创建 `_openid` 升序、`createdAt` 降序组合索引。
- [x] 将数据库集合设为客户端不可读写，并配置已认证用户云函数调用规则。
- [x] 使用“仅创建者可读写”云存储预设权限。
- [x] 接入图片上传、私有图片临时地址、孤儿文件清理和先删图片后删记录流程。
- [x] 实现云端历史分页、所有权校验与用户数据隔离逻辑。
- [x] 保留用户主动进入的离线演示，云端失败不静默降级。
- [x] 上传 `0.2.0` 并配置团队体验版入口，未提交正式审核。

## Phase 3：体验版质量验收（进行中）

- [x] 增加云函数输入、演示模式、文件路径、分页和删除异常自动化测试。
- [x] 增加内置图免上传、孤儿图片清理和离线分页测试。
- [x] 完成 JavaScript、JSON、差异格式检查和敏感信息扫描。
- [x] 整理云环境重复部署手册、阶段总结和发布前清单。
- [ ] 使用两个微信测试账号验证记录、详情、删除和私有图片互相不可访问。
- [ ] 在至少两种 Android 和一种 iOS 真机检查安全区、图片选择、滚动、保存和删除。
- [ ] 完成一次断网、云函数不存在、上传失败、空历史和重复保存的人工验收记录。
- [ ] 核对云函数错误率、调用量、数据库读写量和存储占用。

## Phase 4：真实检测规格冻结（下一阶段，依赖团队输入）

- [ ] 提供代表性原始孔板图像、拍摄参数和经过确认的真实标签数据。
- [ ] 确认 96 孔坐标、分组以及空白、标准和对照孔配置。
- [ ] 确认标准曲线、浓度换算公式、单位、有效范围和模型版本管理办法。
- [ ] 确认污染等级、临界值复测规则、异常样本规则和合规处理建议。
- [ ] 定义真实分析接口的输入、输出、错误码、超时和版本兼容策略。
- [ ] 约定科研验证指标，例如重复性、误差范围、阴阳性对照和人工复核流程。

没有完成本阶段前，不移除 `isDemo: true`，不向用户展示真实检测结论。

## Phase 5：图像与真实分析能力

- [ ] 固化黑箱、摄像头、光照、曝光、距离、方向和文件传输规范。
- [ ] 增加图像方向、孔板定位、曝光、清晰度和完整性质量门禁。
- [ ] 实现孔位提取、背景校正、荧光特征计算和标准曲线换算。
- [ ] 在独立离线数据集上验证算法，记录版本、参数、指标与失败样本。
- [ ] 将真实分析封装为可替换云端实现，通过配置开关与演示分析隔离。
- [ ] 对超时、重复调用和版本升级设计幂等、重试与回滚策略。

## Phase 6：生产化与正式发布

- [ ] 正式发布前升级当前云开发环境，确认套餐有效期和配额。
- [ ] 拆分开发与生产环境，建立最小权限、备份、恢复和变更记录。
- [ ] 配置错误率、调用量、数据库、存储和预算告警。
- [ ] 确认原始图片与检测记录的保存期限、删除政策和用户告知。
- [ ] 完成隐私说明、免责声明、小程序类目和审核材料。
- [ ] 完成双账号隔离、多机型、弱网、兼容性和回归验收。
- [ ] 制定灰度、回滚、数据修复和应急联系人流程后再提交审核。

## 可选后续范围

- [ ] 管理员数据查看与模型配置界面。
- [ ] 检测报告导出、分享和审计记录。
- [ ] 团队硬件与小程序的自动传输联调。

这些功能不属于当前原型范围，只有在 [待团队确认问题](OPEN_QUESTIONS.md) 中确认优先级和权限模型后才能进入实现。
