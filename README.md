# GlowLetter（绿荧来信）

“绿荧来信”是一款面向 iGEM 萘污染荧光检测演示流程的微信小程序。项目采用原生 JavaScript、WXML、WXSS 和微信云开发能力，当前已完成 OpenDesign 原型的界面转换与完整本地演示流程。

> 当前版本是原型演示版。分析结果由本地模拟数据生成，不代表真实实验、污染检测或处理结论。

## 功能概览

- 自定义顶部导航，不使用小程序原生标题栏和返回样式。
- 自定义底部导航：页面中部为毛玻璃悬浮胶囊，触底后与背景融合。
- 首页、历史、我的采用常驻单页壳层，切换时保留各自滚动位置。
- 业务流程页面使用统一沉浸式转场，减少新页面首帧闪白。
- 内置演示图片和模拟结果，无云环境也能走通主要流程。
- 支持创建、查看和删除本地演示记录。

## 快速开始

### 环境要求

- 微信开发者工具
- 可访问本项目小程序 AppID 的开发者账号
- Git

项目没有 npm 依赖，不需要执行依赖安装或构建命令。

### 导入与运行

1. 克隆仓库并打开微信开发者工具。
2. 选择“导入项目”，项目目录指向仓库根目录。
3. 开发者工具会读取 `project.config.json`，并使用 `miniprogram/` 作为小程序目录。
4. 点击“编译”，从启动页进入首页。
5. 点击“开始检测”，可使用内置演示图走完检测、结果保存和历史详情流程。

如果当前账号没有项目 AppID 权限，请联系小程序管理员添加开发成员。个人开发者工具设置会写入被忽略的 `project.private.config.json`，不要把本地配置加入版本控制。

## 项目结构

```text
GlowLetter/
├─ miniprogram/               当前可运行的小程序代码
│  ├─ assets/                 已发布的本地图片资源
│  ├─ components/             自定义顶部栏、底栏和路由转场
│  ├─ pages/                  8 个注册页面、10 个产品视图
│  ├─ styles/                 公共 WXSS
│  └─ utils/                  布局、导航、本地存储和底栏适配
├─ docs/                      架构、协作、计划和待确认事项
├─ docx/PRD.md                产品需求基准
├─ opendesign/                本地原型参考，已被 Git 忽略
├─ Agent.md                   项目开发规范
└─ project.config.json        微信开发者工具共享配置
```

页面与运行时结构详见 [架构说明](docs/ARCHITECTURE.md)。

## 开发约定

- 产品范围以 [PRD](docx/PRD.md) 为准，未确认的科学模型和阈值不得自行补全。
- 所有模拟结果必须保留 `isDemo: true`，并在界面显示演示标识。
- 首页、历史、我的只能通过主页面本地状态切换，不要恢复成原生 `tabBar` 或三个独立路由。
- 页面路由统一通过 `miniprogram/utils/navigation.js` 调用。
- 阶段性修改使用 Conventional Commits，标题用中文描述具体结果，例如：`docs: 完善团队开发与项目架构说明`。
- OpenDesign 源文件、开发者工具私有配置、密钥和云环境私有信息不得提交。

完整流程见 [协作指南](docs/CONTRIBUTING.md) 和 [Agent.md](Agent.md)。

## 提交前检查

```powershell
Get-ChildItem miniprogram -Recurse -Filter *.js |
  ForEach-Object { node --check $_.FullName }

git diff --check
```

此外应在微信开发者工具中至少验证：首页、历史、我的即时切换，检测主流程，详情返回，以及控制台无项目代码错误。
