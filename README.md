# GlowLetter（绿荧来信）

“绿荧来信”是一款面向 iGEM 萘污染荧光检测流程的微信小程序。项目使用原生 JavaScript、WXML、WXSS 与微信云开发，当前已经完成前端原型、云端访客身份、演示分析、私有图片和个人历史记录，并部署为 `0.2.0` 体验版。

> 当前版本是原型演示版。所有分析结果均由服务端固定演示逻辑生成并带有 `isDemo: true`，不代表真实实验、污染检测或处理结论。

## 已实现能力

- 使用自定义顶部导航与常驻单页主入口，首页、历史、我的切换时保留滚动位置。
- 云端初始化失败时展示重试入口，只有用户主动选择后才进入离线演示，不会静默降级。
- 真实选择的图片在保存时上传云存储；内置演示图继续使用包内资源。
- 云端历史支持分页、刷新、详情和删除；私有图片在详情页按需换取临时地址。
- 页面不直接访问数据库，身份、分析、记录与存储统一封装在 `services/` 和 `adapters/`。
- 云函数只信任服务端取得的 OpenID，并校验输入、所有权、分页和云文件路径。
- 本地缓存只用于草稿、页面间待展示结果和用户明确选择的离线演示记录。

## 快速开始

### 环境要求

- 微信开发者工具
- 具备本项目小程序开发权限的微信账号
- Git

小程序端没有 npm 依赖，无需额外构建。云函数依赖由微信云端在部署时安装。

### 导入与运行

1. 克隆仓库，在微信开发者工具中导入仓库根目录。
2. 开发者工具读取 `project.config.json`，以 `miniprogram/` 为小程序目录、`cloudfunctions/` 为云函数目录。
3. 选择团队现有云开发环境；环境 ID 不写入代码或仓库。
4. 点击“编译”，从启动页进入首页。
5. 使用内置演示图或选择本地图片完成分析、保存、历史、详情和删除流程。

本机的开发者工具设置会写入已忽略的 `project.private.config.json`，不得加入版本控制。云资源的重复部署步骤见[云开发部署手册](docs/CLOUD_DEPLOYMENT.md)。

## 开发约定

- 产品范围以 [PRD](docs/PRD.md) 为准，未确认的科学模型、公式和阈值不得自行补全。
- 所有模拟结果必须保留 `isDemo: true`，并在界面中明确显示演示标识。
- 页面路由统一通过 `miniprogram/utils/navigation.js` 调用。
- 页面不得直接访问云数据库；云能力必须经 `miniprogram/services/` 封装。
- 环境 ID、密钥、API Key、账号信息和开发者工具私有配置不得提交。
- 提交信息遵循 Conventional Commits，例如 `docs: 更新原型部署总结与后续计划`。

完整约定见[协作指南](docs/CONTRIBUTING.md)和 [Agent.md](Agent.md)。
